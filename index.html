PUNNARAJ Public Mutual Understanding

This is a public, low-sensitivity working page for shared understanding between PUNNARAJ humans, agents, and future readers.

What this is

A simple public scaffold. It is not ZERO, not the root of PUNNARAJ, and not a final doctrine.

It exists so the current working meanings can be inspected, revised, linked, and rebuilt without custom infrastructure.

Minimum record rule

id:
created:

id gives a record an address. created gives it a place in history.

Additional fields should emerge only when repeated use proves that they solve a real problem.

Current working principles

  • Knowledge is a reusable method, not merely stored data.
  • Wisdom is the ability to evaluate links, context, consequence, and uncertainty.
  • AI should manage structure, provenance, files, and ambiguity; humans remain responsible for real-world meaning.
  • Boundaries should manage sensitivity and blast radius, not freeze useful work.
  • Templates should make patterns visible without becoming rigid law.
  • Every added field, rule, service, or dependency must justify its cost.
  • Use provider ledgers when they already work; build only the missing meaning, relation, and rebuild layers.
